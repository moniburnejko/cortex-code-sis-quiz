## disclaimer - accuracy of questions and AI answers

> **AI-generated content:** questions and explanations generated by AI (via Cortex AI functions) may be inaccurate. LLM models are not infallible - Snowflake exposes older models through AI functions, which are more prone to hallucinations and outdated answers than the latest frontier models. treat AI explanations as a starting point, not a final answer.

> **CSV question bank:** the 1.1K questions in `quiz_questions.csv` are community-sourced from examprepper, not official Snowflake material. answers may not be 100% correct.

> **good habit:** when something looks off, check the Snowflake docs. the documentation is the authoritative source and verifying it is a skill worth building.

> **performance:** if you enable AI-generated questions or AI explanations in the quiz home screen, the app will run noticeably slower. every question requires a live LLM call through Cortex AI functions, and inference takes time. this is determined by the model configured in `docs/AGENTS.md`. you can switch to a faster model (see `docs/custom_config.md`) at the cost of potentially more hallucinations. your call.

## before you start - what is this project?

you are going to use **Cortex Code** - an AI coding agent built into Snowsight - to build a quiz app that runs inside Snowflake. you will not write code manually. instead, you direct the agent using structured prompts, review what it produces, and run the verification checks listed in each phase prompt (`docs/prompts.md`).

by the end you will have:
- a working **Streamlit in Snowflake** app with community-sourced SnowPro Core practice questions
- hands-on experience with **Cortex AI functions** (AI_COMPLETE, AI_PARSE_DOCUMENT)
- a personal study tool to use while preparing for the **COF-C02 certification exam**

see `docs/skills.md` for a description of each skill used in the project.

estimated time: 1-2 hours (depending on your familiarity with Snowflake).

> **Cortex Code docs:** https://docs.snowflake.com/en/user-guide/cortex-code/cortex-code-snowsight

## step 1 - set up the Snowflake environment

> **what this does:** creates a dedicated role (CORTEXADMIN), a warehouse, a database, a schema, and grants all permissions needed for the project.

open `setup/setup.sql` and run each SQL block in Snowsight as ACCOUNTADMIN / SECURITYADMIN / SYSADMIN.
see `setup/setup.md` for explanations of why each object is created and what each grant does.

> **critical — run this first, as ACCOUNTADMIN:**
> ```sql
> ALTER ACCOUNT SET CORTEX_ENABLED_CROSS_REGION = 'AWS_US';
> ```
> this enables cross-region inference, which routes AI function calls to AWS US where Claude models are available. without it, every call to AI_COMPLETE or AI_PARSE_DOCUMENT will fail with `This account is not allowed to access this endpoint` — and the app cannot be built at all. this is step 1 in `setup/setup.sql` for this reason.
>
> the remaining steps (role, warehouse, database, schema) can be run as SYSADMIN.

when done, run the verification query at the end of `setup/setup.sql`:
```sql
SELECT CURRENT_ROLE(), CURRENT_WAREHOUSE(), CURRENT_DATABASE(), CURRENT_SCHEMA();
```
all four should return your expected values.

> **relevant docs:**
> - [access control privileges](https://docs.snowflake.com/en/user-guide/security-access-control-privileges)
> - [Cortex AI Functions - access control](https://docs.snowflake.com/en/user-guide/snowflake-cortex/aisql)

## step 4 - load context into Cortex Code

> **what this does:** gives Cortex Code the information it needs to work in your environment. AGENTS.md is the main context file — without it, the agent will not know your database, schema, or how the app should be built. the skills folder adds custom slash commands used during the build phases.

open a workspace in Snowsight and open the Cortex Code chat panel (white star icon, bottom-right corner).

1. **add AGENTS.md to the workspace:** from your local clone
2. **upload skills:** click the **+** icon in the chat input > **Upload skill folder(s)** > select the `skills/` folder from your local clone

> Cortex Code will now read AGENTS.md as background context on every message you send.
> the skills folder adds custom slash commands (`/cortex-prompt`, `/streamlit-in-snowflake`, `/cortex-ai`, `/switch-exam`) that Cortex Code uses during the build phases. see `docs/skills.md` for details.

## step 5 - phase 1: infrastructure

> **what happens:** Cortex Code creates 3 tables (EXAM_DOMAINS, QUIZ_QUESTIONS, QUIZ_REVIEW_LOG) and 2 stages. then it reads the official SnowPro study guide PDF using AI_PARSE_DOCUMENT and extracts the 6 exam domains using AI_COMPLETE.

1. open `docs/prompts.md`, copy the entire **phase 1** block, paste it into the Cortex Code chat
2. Cortex Code will set up the infrastructure and then ask you to upload the PDF

**upload SnowProCoreStudyGuide.pdf to STAGE_QUIZ_DATA:**
- Snowsight: **Data > Add Data > Load files to a stage**
- select stage STAGE_QUIZ_DATA in your database and schema
- upload the file, go back to Cortex Code and type: `file uploaded`

3. Cortex Code extracts the 6 exam domains and populates EXAM_DOMAINS

> **relevant docs:**
> - [AI_PARSE_DOCUMENT](https://docs.snowflake.com/en/sql-reference/functions/ai_parse_document)
> - [AI_COMPLETE](https://docs.snowflake.com/en/sql-reference/functions/ai_complete-snowflake-cortex)

## step 6 - phase 2: load questions

> **what happens:** Cortex Code creates a CSV file format (FF_CSV) and uses COPY INTO to load questions from quiz_questions.csv into QUIZ_QUESTIONS. then it backfills domain_name from EXAM_DOMAINS.

1. copy the **phase 2** block from `docs/prompts.md` and paste it into the chat
2. Cortex Code will ask you to upload the CSV file

**upload quiz_questions.csv to STAGE_QUIZ_DATA:**
- Snowsight: **Data > Add Data > Load files to a stage**
- the file is in the `data/` folder of the repo - upload it to STAGE_QUIZ_DATA
- return to Cortex Code and confirm

---

## step 7 - phase 3: build and deploy the app

> **what happens:** Cortex Code reads the full app specification from AGENTS.md and writes quiz.py - a complete Streamlit in Snowflake app. it then runs a pre-deploy scan to catch known SiS issues, and deploys the app using CREATE OR REPLACE STREAMLIT.

1. copy the **phase 3** block from `docs/prompts.md` and paste it into the chat
2. wait - this is the longest phase. Cortex Code will write the full app, then scan it for issues, then fix any problems, then ask you to upload the files

**when Cortex Code asks you to upload files to STAGE_SIS_APP:**
- `quiz.py` 
- `environment.yml` 

download both files from the workspace.

**upload both files to STAGE_SIS_APP:**
- Snowsight: **Data > Add Data > Load files to a stage** > select STAGE_SIS_APP
- upload quiz.py and environment.yml
- return to Cortex Code and confirm

3. Cortex Code runs CREATE OR REPLACE STREAMLIT to deploy the app

> **relevant docs:**
> - [Streamlit in Snowflake](https://docs.snowflake.com/en/developer-guide/streamlit/about-streamlit)
> - [CREATE STREAMLIT](https://docs.snowflake.com/en/sql-reference/sql/create-streamlit)

## troubleshooting

**app does not load or crashes:**
use the **fix session** prompt from `docs/prompts.md`. describe the error you see, paste the prompt into Cortex Code, and let it diagnose. once fixed, re-upload quiz.py to STAGE_SIS_APP and redeploy.
see `docs/known-bugs.md` for common issues and solutions.

**Cortex Code gives wrong results or ignores AGENTS.md:**
make sure AGENTS.md is added to the session context (attachment icon). if the session has been running for a long time, start a new session and re-add the file.

**model availability error on AI_COMPLETE:**
run `/cortex-ai` in Cortex Code - it will diagnose which step fails and provide the fix.
most common cause: cross-region inference is not enabled. run step 1 from `setup/setup.sql` as ACCOUNTADMIN.

**COPY INTO loads 0 rows:**
check that the file was uploaded to the correct stage (STAGE_QUIZ_DATA, not STAGE_SIS_APP).
